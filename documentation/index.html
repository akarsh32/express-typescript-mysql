Installation
  1. Simply run
    $ npm install

or via Yarn.

$ yarn
  1. Edit the config file depends on your server environment
    module.exports = {
     MySQL_CON :{
     connectionLimit : 10,
     host: 'xx.xx.xx.xx',
     user: 'root',
     password: '',
     database: 'restapi',
     debug: false,
     Promise: require('bluebird')
     },
     PORT: 3307,
     HOST : 'localhost'
}



3. use the dump file to populate data or call the API to seed the database.

# Running in development
```bash
$ npm run dev

Type rs inside the terminal to manually restart the app.

more information ts-node-dev

Production

 
$ npm run build

This will build all files inside dist folder and run pm2 or via AWS Elastic Beanstalk.

Linting

 
$ npm run lint

Automated Tests

In order for this to work, you need to update your testing config json file.

$ npm run test

If you want to test via Postman, import Rest API.postman_collection.json to Postman

#Testing Via cURL

Paginate Data

 
curl -X GET --url 'http://localhost:3000/persons/page/0'

Get Person with id = 2

 
curl -X GET --url 'http://localhost:3000/person/2'

Get Person's contacts with id = 4

 
curl -X GET --url 'http://localhost:3000/person/contacts/4'

Create Person

 
curl -X POST \
--url 'http://localhost:3000/person'  \
-H 'Content-type: application/json' \

--data-raw '{
"name": "aa",
"surname": "sur",
"email": "aa@asd.com",
"phone": "09989876543",
"gender": "male"
}'

Update Person

 
curl -X PUT \
--url 'http://localhost:3000/person'  \
-H 'Content-type: application/json' \

--data-raw '{
"id": 4,
"name": "aa5",
"surname": "sur1",
"email": "aa@asd1.com",
"phone": "09989876543"
}'

Delete

 
curl -X DELETE \
--url 'http://localhost:3000/person/1'  \
-H 'Content-type: application/json' \

--data-raw '{
"id": 1,
"name": "aa1",
"surname": "sur1",
"email": "aa@asd1.com",
"phone": "09989876543",
"gender": "female"
}'
